# UI-20 Create Memory Screen Polish — 設計メモ

## 主要変更点

### 1. カスタムヘッダーに変更

**旧実装**: `ScreenHeader` コンポーネント（サブタイトルなし）
**新実装**: インラインカスタムヘッダー

- タイトル: 「思い出ノートを作る」
- サブタイトル: 「写真を選ぶだけで、場所・流れ・日記をまとめます」
- 戻るボタン: ivory ラウンドボタン（既存 ScreenHeader と同様の動作）

理由: ScreenHeader はサブタイトルをサポートしていないため。画面専用のため shared component への変更を避けた。

### 2. 写真セクションを大型ヒーローカードに変更

**旧実装**:
- 写真なし: テキスト + ボタン（hero 部分は別セクション）
- 写真あり: 横スクロールサムネイル + 追加ボタン

**新実装**:
- 写真なし: 高さ 220px の大きなタップ可能カード（破線ボーダー）
  - ivory 背景 + 📷 アイコン + 「写真を選ぶ」テキスト + 「カメラロールから選ぶ」ボタン（カード内部）
  - カード全体がタップ可能
- 写真あり: カバーカード形式
  - 先頭写真を 200px 全幅で大きく表示（cover）
  - 「📷 N枚選択中」バッジをオーバーレイ
  - 2枚目以降を横スクロールストリップで表示
  - 「＋ 写真を追加する」ボタン（secondary スタイル）

### 3. Auto Organize セクションを追加（常時表示）

**旧実装**: 「作成の流れ」3ステップカード（写真未選択時のみ）
**新実装**: 「作成後に自動で整理されます」セクション（常時表示）

内容:
1. 🗺️ 場所と流れを整理
2. ✏️ AI日記の下書き
3. 📤 共有カードを作成

理由: ユーザーが写真を選んだ後も「何が自動でされるか」が分かるようにする（スペック: "auto grouping preview, intelligence reassurance block"）。

### 4. フォームのラベル変更

- タイトル: 「タイトル *」→ 「タイトル」（* 必須マーク削除）
- メモ: 「メモ」→ 「メモ（省略可）」
- 共有ノートの説明: 「Phase 11 以降」→ 「ノート作成後に設定」

### 5. 保存ボタン文言変更

「ノートを作成する」→ 「思い出ノートを作成」

### 6. Loading states 改善

- ノート作成中: `ActivityIndicator` + 「思い出ノートを作成しています...」（横並び）
- アップロード中: 既存プログレスバー（変更なし）
- ヒントテキスト: 3パターン（タイトル未入力 / 写真あり / 写真なし）

## 変更しなかったもの

- `useCreateNote` hook — API変更なし
- `usePhotoPicker` hook — API変更なし
- `usePhotoUpload` hook — API変更なし
- `handleSave` ロジック — 変更なし
- 作成後の遷移 `router.replace(/(app)/notes/${noteId})` — 変更なし
- タイトル必須バリデーション — 変更なし（`canCreate = !!title.trim() && !isProcessing`）

## タイトル必須の判断

`useCreateNote.validate()` がタイトル必須で実装されている。
フォールバック（「無題の思い出」）を適用するには hook の bypassing が必要で、
複雑度が上がるため今回は title 必須を維持した。
代わりにヒントテキスト（「タイトルを入力するとノートを作成できます」）で
ユーザーを誘導する。
