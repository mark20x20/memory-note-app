# Phase 7: 既知の問題・制限事項

## 1. Storage read は owner のみ（メンバーはアクセス不可）

**内容**: Storage Rules では `request.auth.uid == uid` のみを許可しており、共有ノートのメンバーは他ユーザーの写真 URL にアクセスできない。

**影響**: Phase 7 時点では共有ノート機能未実装のため実害なし。

**対応予定**: Phase 11（コラボレーション実装）で Storage Rules を拡張。

---

## 2. coverPhotoURL の非正規化による不整合リスク

**内容**: 写真を削除した際に `memory_notes/{noteId}.coverPhotoURL` が自動更新されない。先頭写真を削除しても古い URL が残存する可能性がある。

**影響**: Home 画面カードに削除済み写真の URL が表示され、画像読み込みエラーが発生する可能性。

**対応予定**: Phase 10（写真削除機能実装）で写真削除時に coverPhotoURL を更新するロジックを追加。

---

## 3. 並列アップロード中の部分失敗

**内容**: `Promise.all` で並列アップロードを行うため、一部の写真が成功・一部が失敗した場合でも全体を失敗扱いにする。

**影響**: 5枚中4枚成功・1枚失敗でもノートの `photoCount` は更新されず 0 のまま。

**対応予定**: Phase 10 でリトライ UI や部分成功ハンドリングを検討。

---

## 4. 画像圧縮未対応

**内容**: Phase 7 では `expo-image-manipulator` による圧縮を行わず、オリジナル画像をそのままアップロードする。

**影響**: 大容量写真（〜10MB）の場合、アップロード時間が長くなる可能性。

**対応予定**: Phase 10 以降で圧縮・リサイズを導入予定。

---

## 5. useNotePhotos の再接続ハンドリング未実装

**内容**: `onSnapshot` の接続が切れた場合（オフライン復帰など）の自動再接続ロジックを実装していない。

**影響**: ネットワーク切断後の写真リストが更新されない可能性。

**対応予定**: Phase 10 または Phase 11 でオフライン対応と合わせて実装。
