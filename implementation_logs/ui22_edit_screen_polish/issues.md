# UI-22 Edit Screen Polish — 課題・既知の問題

## 解決済み

### #1 ScreenHeader の非 warm デザイン
- **問題**: `ScreenHeader title="編集"` はボーダーありで他画面との統一感がなかった
- **解決**: カスタムヘッダー（タイトル + サブタイトル + プレビューボタン）に置き換え

### #2 OverviewPanel の Alert テキストが UI-16B セマンティクスと不一致
- **問題**: "共有ノートに変更すると" という表現は、noteType が即変更されるような誤解を招く
- **解決**: "メンバーを招待すると、このノートが共有ノートになります。" に変更

### #3 PhotosPanel に開発者向けメモが残存
- **問題**: "並び替えはUI-3で実装予定です。" がユーザーに表示されていた
- **解決**: テキストと関連スタイルを削除

### #4 FlowsPanel / PlacesPanel の空状態が命令形
- **問題**: 空状態が「〇〇してください」というアクション要求になっており、処理待ちケースに不適
- **解決**: "整理中です" + 処理内容の説明文に変更

## 残存課題

### #5 MemoPanel の minor warm 改善（未対応）
- **内容**: MemoPanel は既に4状態実装済みで機能的に問題なし
- **判断**: スタイル微調整は別タスクで対応（現状で十分な品質）

### #6 FlowsPanel 空状態 — フロー "0件" vs "処理待ち" の区別
- **内容**: `groups.length === 0` だけでは「処理後0件」と「処理前」の区別ができない
- **対応案**: `note.placeEnrichmentStatus` を FlowsPanel に渡して分岐
- **判断**: 現状は warm な "整理中" テキストで十分。必要に応じて別タスクで対応
