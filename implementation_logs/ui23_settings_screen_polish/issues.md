# UI-23 Settings Screen Polish — 課題・既知の問題

## 解決済み

### #1 ScreenHeader の非 warm デザイン
- **問題**: `ScreenHeader title="設定"` はボーダーありで他画面との統一感がなかった
- **解決**: カスタムヘッダー（タイトル + サブタイトル + 対称スペーサー）に置き換え

### #2 Profile Card の fallback 文言が不親切
- **問題**: displayName/email が未設定のとき "未設定" と表示していた
- **解決**: "思い出ノートユーザー" / "メールアドレス未設定" に変更

### #3 未実装リンクが押しても何も起きない
- **問題**: 利用規約・プライバシーポリシー・お問い合わせが未実装のまま `›` を表示
- **解決**: `comingSoon` prop で "準備中" バッジを表示、disabled に設定

### #4 バージョン表記に開発者向け文言
- **問題**: "Memory Note v1.0 (Phase 4)" とユーザーに表示されていた
- **解決**: "Memory Note v1.0" に変更

### #5 Alert テキストが簡素すぎた
- **問題**: "ログアウト" / "ログアウトしますか？" という最小限の文言
- **解決**: "ログアウトしますか？" / "この端末からアカウントをログアウトします。" に改善

## 残存課題

### #6 利用規約 / プライバシーポリシー / お問い合わせの実リンク未実装
- **内容**: 現在すべて "準備中" バッジで disabled
- **対応**: 実URLが決定したら `Linking.openURL` または WebView で実装
- **判断**: UI-23 の scope 外。別タスクで対応

### #7 プロフィール編集機能なし
- **内容**: displayName / email / photoURL の変更 UI は未実装
- **対応**: Firebase Auth updateProfile を使ったプロフィール編集画面
- **判断**: UI-23 の scope 外。別タスクで対応

### #8 権限設定（写真・位置情報）の移動
- **内容**: 旧 "プライバシー・権限" セクションを「アプリ」に統合したため、
  権限設定へのリンクは "アプリ" セクションには含めなかった
- **影響**: iOS の「設定」アプリへのリンクが削除された
- **判断**: 権限はシステム設定アプリで管理するのが標準。別タスクで必要なら復活
