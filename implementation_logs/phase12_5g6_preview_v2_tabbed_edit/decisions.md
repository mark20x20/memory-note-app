# Phase 12.5G-6 Decisions

## プレビューを「確認画面」ではなく「思い出を見る画面」に寄せる

`preview v2` では、以下のようなシステム都合の文脈を前面に出さない方針にした。

- 生成しました
- 整理しました
- 推定しました
- confidence
- checklist
- review / validation

これらはアプリ内部では重要でも、ユーザーが思い出を開く体験には不要なノイズになりやすい。

## 編集導線だけを静かに残す

プレビューで唯一残す操作は、基本的に `編集する` への導線とする。

理由:
- 思い出の閲覧体験を壊さない
- 何か違和感がある時だけ編集へ移れる
- 確認作業を強制しない

## 編集画面は「長い1画面」より「責務ごとのタブ」が適している

従来案は一画面の中に:
- タイトル
- 写真
- flow
- 場所
- メモ

がすべて見えており、少し細かすぎて圧を感じやすい。

そのため、編集は以下のタブに分割する方針に変更した。

- `概要`
- `写真`
- `流れ`
- `場所`
- `メモ`

## 深い操作はタブ内でだけ見せる

`flow を分ける / 結合する / 写真を移す / 候補から場所を選ぶ` などの深い操作は、
常に見せるのではなく、対応するタブを開いた時だけ見せる。

## Preview / Edit の責務を明確化する

- Preview = 思い出を味わう
- Edit = 必要なところを整える
