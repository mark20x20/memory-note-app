# Release Feature Scope v2

## 1. 機能分類の前提

本分類は、**Release v1 を実際に公開できる本番アプリ**として扱い、その後の改善を段階分けしたものです。  
中心軸は変えません。

> **写真を選ぶだけで、友人・家族・恋人との思い出を、地図付きノートとして共同で残し、SNSにも美しく共有できるアプリ。**

### フェーズ定義

| 分類 | 定義 |
|---|---|
| Release v1 | 初期公開版に必ず入れる機能 |
| Release v1.1 | 公開後すぐの改善・軽微な拡張 |
| Phase 2 | 重要だが初期公開には必須でない機能 |
| Phase 3 | 大きな体験拡張・開発負荷が高い機能 |
| Future | 長期的な構想 |
| Not Planned | 作らない、または作るべきでない機能 |

## 2. Release v1 に入れる機能

### 2.1 アカウント・認証

| 機能 | 内容 | 理由 |
|---|---|---|
| アカウント登録 | メール、Apple、Google 等で登録できる | 共有・クラウド保存の前提 |
| ログイン | 再利用・機種変更・複数端末利用に対応 | 実運用に必須 |
| ログアウト | 安全なセッション終了 | 基本機能 |
| プロフィール作成 | 表示名、アイコン、簡易プロフィール | 共有ノートで必要 |
| アカウント削除 | データ削除導線を含む | プライバシー必須 |

### 2.2 クラウド保存・同期

| 機能 | 内容 | 理由 |
|---|---|---|
| クラウドDB | ノート、メンバー、権限、メタ情報を保存 | v1の中核 |
| クラウドストレージ | 写真を保存する | 共有ノートの前提 |
| 基本同期 | アップロード済みデータを複数端末で参照 | 実用性確保 |
| データ整合性管理 | オーナー変更、削除、参照整合を管理 | 共同利用の基盤 |
| 削除反映 | ノート削除時に関連データ削除を行う | 信頼性と法務対応 |

### 2.3 写真アップロード・画像管理

| 機能 | 内容 | 理由 |
|---|---|---|
| 写真選択 | OS標準のピッカー等で選択 | 入口機能 |
| 写真アップロード | 選択した写真をクラウドへ送信 | v1必須 |
| EXIF情報読み取り | 撮影日時、GPS、向き等を取得 | 自動整理の基礎 |
| サムネイル生成 | 表示用軽量画像を作成 | UXとコスト最適化 |
| 画像圧縮 | 表示用の圧縮保存を行う | コスト抑制 |
| GPSなし写真対応 | 位置情報がなくても保存できる | 実利用に必須 |
| 写真削除 | ノートからの画像削除 | 管理とプライバシー対応 |

### 2.4 思い出ノート作成

| 機能 | 内容 | 理由 |
|---|---|---|
| 時系列整理 | 写真を撮影日時順に並べる | 一日の流れを作る |
| 場所別グループ化 | 近い時間・位置でまとめる | スポット単位で見やすい |
| ノート作成 | 1つの思い出単位で保存 | サービスの核 |
| タイトル入力・編集 | ユーザーが自由に変更できる | AI補助前提 |
| 本文入力・編集 | 短文メモを追記できる | 思い出の文脈化 |
| 保存前プレビュー | 生成結果を確認してから保存 | 誤保存防止 |
| ノート編集 | 保存後も修正可能 | 実用性確保 |
| ノート削除 | 単位ごとに削除可能 | 必須の管理機能 |

### 2.5 AI生成

| 機能 | 内容 | 理由 |
|---|---|---|
| AIタイトル生成 | 日付・場所・写真情報から提案 | ノート化を支援 |
| AI短文日記生成 | 写真から短い説明文を生成 | 中心価値 |
| AI生成の手動修正 | 生成文を編集可能にする | 精度担保 |
| AI失敗時の代替文生成 | ルールベース等でフォールバック | 運用品質の確保 |

### 2.6 地図・位置情報

| 機能 | 内容 | 理由 |
|---|---|---|
| 地図ピン表示 | 写真位置を地図上に表示 | 地図付きノートの核 |
| 場所名推定 | 緯度経度から地点名・エリア名を推定 | 読みやすさ向上 |
| 地図上の閲覧 | ノート詳細で場所を確認できる | 振り返り体験 |
| 位置情報なし時の代替表示 | 日時のみ、または「場所情報なし」表示 | 離脱防止 |

### 2.7 閲覧・整理

| 機能 | 内容 | 理由 |
|---|---|---|
| ノート一覧 | 作成済みノートを一覧表示 | 基本導線 |
| ノート詳細 | 写真、地図、本文、メンバーを表示 | 中心画面 |
| カレンダー表示 | 日付ベースで振り返る | 使いやすさ向上 |
| タグ | 最低限の分類タグを付与 | 後の整理を見据える |
| 検索 | タイトル・場所名・本文の簡易検索 | ノート増加時に有効 |
| お気に入り | 大事なノートを固定 | v1でも軽量で有効 |
| 月次・年次サマリー | 一覧で簡易に振り返る | 継続利用を支える |

## 10. スコープ肥大化を防ぐルール

| ルール | 内容 |
|---|---|
| 写真起点を崩さない | 文章入力起点、SNS起点にしない |
| 公開SNS化しない | フィード、いいね、コメントは入れない |
| 地図をナビ化しない | 道案内、ルート検索、口コミを作らない |
| 旅行予約領域に入らない | 航空券・ホテル・決済を作らない |
| 常時GPSを要求しない | EXIF中心で扱う |
| 写真バックアップアプリにしない | Google Photos代替を狙わない |
| 共同編集は権限を簡潔に保つ | Owner / Editor / Viewer に固定する |
| 共有カードは美しく、重くしすぎない | 出力機能に集中する |
| AIは補助に留める | AIチャットや長文生成は後回し |
| プライバシー説明を削らない | 削除、同意、公開範囲は必須 |
| コストが増える外部APIは差し替え可能にする | 将来の運用変更に備える |

## 決定事項

- **Release v1 からクラウド前提**にする
- **Release v1 からアカウント必須または強く推奨**にする
- **Release v1 から共有ノートと共同編集**を入れる
- **Release v1 から Owner / Editor / Viewer を採用**する
- **Release v1 から SNS共有カード生成を必須**にする
- **1:1 / 4:5 / 9:16 のカード比率**を対象にする
- **Instagram / LINE / X / Facebook の共有導線**を入れる
- **アプリ内SNSフィード、いいね、コメント、フォローは入れない**
- **本格ナビ、飲食店口コミ、旅行予約、常時GPSトラッキングは入れない**

## 未決定事項

- クラウド基盤の最終選定
- 画像原本をクラウドにどこまで保存するか
- AIに送るデータの最小範囲
- 共有リンクの公開範囲
- 削除時のメンバー・画像・ノートの厳密な扱い
- SNS共有カードのテンプレ最終仕様

## 不足している証拠

- 共有ノート機能の実際の利用意向
- 共同編集の許容範囲
- SNS共有カード経由の流入効果
- 日本市場での「家族・恋人・友人との共同記録」需要の定量データ

## 主要リスク

- 共同編集と権限管理で設計が複雑化する
- クラウド保存と画像保存でコストが増える
- 共有仕様が曖昧だとプライバシー事故につながる
- SNS共有カードが重くなりすぎると初期開発が遅延する

必要であれば次に、**このスコープをもとにした「Must / Should / Could / Won’t」形式の実装優先度表**に落とし込めます。
