# SNS Share Card Spec v1
Memory Note App / 思い出ノートアプリ  
Release v1 用仕様

---

## 1. 目的

SNS共有カードは、アプリ内の「思い出ノート」を**外部SNSで見栄えよく共有するための画像出力機能**です。  
Release v1 では以下を同時に満たします。

- 思い出の内容が一目で伝わる
- Instagram / LINE / X / Facebook で使いやすい
- アプリの認知導線になる
- ただし、**位置情報・個人情報の漏えいを防ぐ**
- 共有前の**プレビューを必須**にする
- アプリ内SNS化はしない

### 役割の定義
SNS共有カードは「投稿用の完成画像」であり、ノート本文そのものではありません。  
そのため、ノートの閲覧・編集権限とは別に、**外部共有のための表現物**として扱います。

---

## 2. 共有カードの役割

### 2.1 主要な役割
- ユーザーの思い出を美しく要約する
- ノートの存在を外部に伝える
- SNS投稿のハードルを下げる
- アプリのブランド認知につなげる

### 2.2 非目的
- アプリ内での公開フィード
- いいね、コメント、フォロー
- 投稿一覧のSNS化
- 口コミ投稿の代替
- 詳細な位置公開

---

## 3. 対応フォーマット

Release v1 では以下の3形式を必須対応とします。

| フォーマット | 比率 | 推奨px | 主な用途 | Release v1優先度 |
|---|---:|---:|---|---|
| Square | 1:1 | 1080 × 1080 | X、汎用共有、一覧的な見せ方 | P0 |
| Feed | 4:5 | 1080 × 1350 | Instagram投稿の主力 | P0 |
| Story | 9:16 | 1080 × 1920 | Instagramストーリー、縦長共有 | P0 |

### 実装ルール
- いずれも**高解像度静止画像**として生成する
- フォーマットごとにレイアウト最適化する
- 単純なリサイズではなく、**余白・文字量・要素配置を別設計**にする
- 1つのノートから複数フォーマットを生成可能にする

---

## 4. カードに表示する要素

### 4.1 必須表示要素
| 要素 | 表示可否 | 役割 | 備考 |
|---|---|---|---|
| メイン写真 | 必須 | 主視覚要素 | 1枚を主役にする |
| タイトル | 必須 | 内容の要約 | 短く読みやすく |
| 日付 | 必須 | いつの思い出か明示 | `2026.06.07` など |
| 場所名 | 必須 | どこでの出来事か明示 | 地名 or 施設名 |
| 短文日記 | 必須 | 感情や流れを伝える | 1〜2文程度 |
| 写真枚数 | 必須 | ボリューム感 | `12 photos` ではなく日本語表記も可 |
| スポット数 | 必須 | ルート感・行程感 | `3スポット` 等 |
| ミニマップ | 必須 | 地図付きノートの特徴表現 | 詳細座標は出さない |
| ルート感 | 必須 | 移動の流れを示す | 線や点で表現 |
| アプリロゴ | 必須 | ブランド認知 | 小さく自然に表示 |
| “Made with Memory Note” | 必須 | ブランド導線 | 強すぎない表記 |

### 4.2 任意表示要素
| 要素 | 優先度 | 条件 |
|---|---:|---|
| スポット名の短縮一覧 | P1 | 文字数が収まる場合 |
| カテゴリタグ | P1 | `旅行`, `カフェ`, `デート` など |
| 一言キャッチ | P1 | タイトルとは別に短い補足が必要な場合 |
| QR / URL | P2 | v1では原則非推奨。必要なら限定利用 |
| 参加人数 | P2 | 共有相手の同意とプライバシーが担保できる場合 |

### 4.3 表示順の基本
基本の優先順位は以下です。

1. メイン写真
2. タイトル
3. 日付
4. 場所名
5. 短文日記
6. ミニマップ / ルート感
7. 写真枚数 / スポット数
8. ブランド要素

## 5. 表示しない / 抑制する要素

Release v1 では、以下を**表示しないか、強く抑制**します。

| 要素 | 方針 | 理由 |
|---|---|---|
| 精密GPS座標 | 非表示 | 住所特定リスク |
| 住所 | 非表示 | 生活圏漏えいリスク |
| 参加者の詳細プロフィール | 非表示 | 個人情報保護 |
| メールアドレス | 非表示 | 不要な個人情報 |
| 招待リンク | 非表示 | 漏えいリスク |
| 非公開メモ全文 | 非表示 | プライバシー事故防止 |
| 他メンバーの個人情報 | 非表示 | 同意なし外部送信を避ける |
| 編集履歴の詳細 | 非表示 | 情報過多・漏えいリスク |
| 位置の正確なピン座標 | 非表示 | 自宅・勤務先推定防止 |

## 6. 位置情報ぼかし仕様

### 6.1 位置表示レベル
共有カードでは、以下の4段階で制御します。

| レベル | 表示内容 | 用途 |
|---|---|---|
| exact | 位置情報の詳細表示 | **v1では原則使わない** |
| area | エリア名のみ表示 | 推奨 |
| city | 市区町村レベル表示 | 安全寄り |
| hidden | 位置非表示 | 最も安全 |

### 6.2 Release v1 のデフォルト
- **外部SNS公開用カードのデフォルトは `area` または `city`**
- 精密座標はカード上に出さない
- ノート内部表示と外部共有表示は分離する

### 6.3 ぼかし方法
- 地図は**簡略ミニマップ**にする
- ピンは**厳密位置ではなく近傍表示**にする
- ルート線は**概略線**に留める
- 自宅・勤務先が推測できる表現は避ける

### 6.4 推奨設定
| 共有先 | デフォルト表示 | 理由 |
|---|---|---|
| Instagram / X | city または area | 公開性が高い |
| LINE | area | 友人間共有が主 |
| Facebook | city または area | 公開投稿の可能性がある |

### 6.5 ルール
- 共有カード生成時に、位置情報の公開レベルをユーザーが確認できること
- `exact` を使う場合は、**明示的なオン操作**が必要
- デフォルトは安全側に倒す

## 7. カードテンプレート

Release v1 では最低3種のテンプレートを用意します。

### 7.1 Photo-first
| 項目 | 内容 |
|---|---|
| 特徴 | 写真を大きく見せる王道構成 |
| 向いている用途 | Instagram投稿、旅行、カフェ、デート |
| 表示要素 | 大きいメイン写真、タイトル、日付、場所名、短文、下部にミニマップ |
| 注意点 | 文字を載せすぎない。写真が主役 |

### 7.2 Map-first
| 項目 | 内容 |
|---|---|
| 特徴 | 地図とルート感を強める構成 |
| 向いている用途 | 複数スポット、日帰りお出かけ、旅行記録 |
| 表示要素 | ミニマップ、ルート線、スポット数、写真枚数、タイトル、小さめ写真 |
| 注意点 | 地図を強くしすぎると写真の魅力が落ちる |

### 7.3 Diary-first
| 項目 | 内容 |
|---|---|
| 特徴 | 短文日記と感情を前面に出す構成 |
| 向いている用途 | 記念日、デート、思い出の一言共有 |
| 表示要素 | 短文日記、大きめタイトル、日付、場所名、控えめ写真 |
| 注意点 | 長文は避ける。読める情報量に絞る |

### 7.4 テンプレート選択ルール
- デフォルトは **Photo-first**
- スポット数が多い場合は **Map-first**
- メモが印象的な場合は **Diary-first**
- ユーザーは手動で変更可能
- AIが自動推薦してもよいが、最終決定はユーザー

---

## 8. 生成フロー

### 8.1 フロー全体
1. ノート詳細を開く
2. 共有ボタンを押す
3. フォーマットを選ぶ
4. テンプレートを選ぶ
5. 表示情報を確認する
6. 位置情報ぼかしレベルを確認する
7. プレビューを生成する
8. 画像を書き出す
9. 端末に保存する
10. OS共有シートを開く
11. SNSへ投稿する

### 8.2 詳細
- **ノート詳細** からのみ共有カード生成を開始できる
- 共有対象は、ユーザーが閲覧権限を持つノートに限る
- プレビュー生成前に、位置表示の注意を出す
- 保存後は端末の共有シートへ渡す
- アプリ内で直接SNS投稿はしない

### 8.3 共有先導線
OS共有シートを中心にする。  
個別導線としては以下を補助的に表示してよい。

- 画像を保存
- LINEで共有
- Xで共有
- Instagramに保存して投稿
- Facebookで共有

ただし、各SNSの投稿UIはアプリ内で再実装しない。

## 9. 権限と同意

### 9.1 権限関係
| ロール | 共有カード生成 | 外部保存 | 備考 |
|---|---:|---:|---|
| Owner | 可 | 可 | 最終管理者 |
| Editor | 可 | 可 | 共同編集者 |
| Viewer | 可 | 可 | 閲覧専用でも共有は可能 |

### 9.2 共有カード生成の前提
- ユーザーがそのノートを閲覧できること
- 外部公開プレビューを確認していること
- 位置情報の公開レベルを確認していること

### 9.3 他メンバーの写真
- 他メンバーが追加した写真も、共有カードに使用可能
- ただし、**外部公開前提のため明示的な注意表示**を入れる
- 必要なら「このカードには共有ノート内の写真が含まれます」と示す

### 9.4 プレビュー必須
- **プレビュー表示を省略して直接保存・共有は不可**
- プレビュー画面で位置情報レベルを再確認できること
- ユーザーが確認してから生成確定すること

## 10. 画像生成方式

### 10.1 推奨案
**Cloud Functions側生成を推奨**します。

#### 理由
- テンプレート統一がしやすい
- 端末性能差を吸収しやすい
- フォントやレイアウトの再現性が高い
- 共有画像のキャッシュと管理がしやすい
- 生成失敗時のログ取得がしやすい

### 10.2 モバイル端末側生成
- 可能だが、端末差分が大きい
- フォント、描画、品質の再現性に不利
- v1では補助的、または将来拡張として扱う

### 10.3 保存先
- 生成した共有カードは `share_cards` エンティティに紐づける
- Firebase Storage に保存する
- ノート単位で再利用できるようにする
- 同じ条件の再生成ではキャッシュを使ってもよい

### 10.4 キャッシュ方針
| 項目 | 方針 |
|---|---|
| 同一ノート・同一テンプレ・同一比率 | キャッシュ可能 |
| 位置情報レベル変更後 | 再生成 |
| 写真差し替え後 | 再生成 |
| タイトルや本文変更後 | 再生成 |
| ブランド表記変更後 | 再生成 |

## 11. UI要件

### 11.1 共有カード生成画面
表示するもの:
- 対象ノート名
- フォーマット選択
- テンプレート選択
- 位置情報の公開レベル
- 使用するメイン写真
- 短文日記
- プレビュー生成ボタン

### 11.2 プレビュー画面
表示するもの:
- 実寸に近いカードプレビュー
- 位置情報の表示レベル
- 注意文
- 保存ボタン
- 共有ボタン
- 再編集ボタン

### 11.3 位置情報注意表示
必須文言の要旨:
- このカードは外部SNSで共有される
- 位置情報はぼかし表示または地名表示になる
- 精密な場所は表示されない

### 11.4 保存完了表示
- 画像の保存完了
- 端末内保存先の案内
- 共有シートを開くボタン

### 11.5 共有失敗表示
- 生成失敗
- 保存失敗
- 共有シート失敗
- 再試行ボタン
- 別フォーマットで再生成する導線

## 12. エラーハンドリング

| エラー種別 | 表示 | 対応 |
|---|---|---|
| 画像不足 | メイン写真がない | 写真選択を促す |
| 地図取得失敗 | 地図を生成できない | 地図なしテンプレへ切替 |
| AI短文なし | 文がない | ルールベース文で代替 |
| Storage保存失敗 | 画像を保存できない | 再試行 / 端末保存のみ |
| OS共有失敗 | 共有シート起動失敗 | 端末保存だけ案内 |
| 権限不足 | 共有不可 | 閲覧権限の確認表示 |

### フォールバック方針
- 地図が失敗しても、写真＋タイトルだけでカード生成できる
- AI短文がなくても、固定文で成立させる
- 保存失敗時も、プレビュー画像を一時保存できるとよい
- 生成失敗でノート閲覧や編集は止めない

## 13. ブランド設計

### 13.1 表記
- アプリ名: **Memory Note**
- 表記補助: **Made with Memory Note**

### 13.2 表示位置
- カード下部の端
- もしくは右下の小さなラベル
- 主役を邪魔しない位置に限定

### 13.3 強すぎない宣伝
- ロゴは小さく
- ブランド文言は1行以下
- 写真・タイトルより目立たせない
- 余白の中に自然に入れる

### 13.4 ブランドの役割
- アプリ名を記憶してもらう
- 共有画像が「このアプリで作られた」と分かる
- ただし広告臭を出しすぎない

## 14. 受け入れ基準

Release v1 の受け入れ基準は以下です。

- 1:1 / 4:5 / 9:16 の3形式を生成できる
- 共有前プレビューが必須である
- メイン写真、タイトル、日付、場所名、短文、写真枚数、スポット数を表示できる
- ミニマップまたはルート感を表現できる
- 精密GPS、住所、招待リンク、個人情報を出さない
- 端末保存とOS共有シートが使える
- Instagram / LINE / X / Facebook の導線が成立する
- Owner / Editor / Viewer いずれでも権限内で共有画像を生成できる
- 地図失敗時でもフォールバックカードを作れる
- 共有カードがアプリ内SNS化の導線になりすぎない

## 決定事項
- Release v1 では 1:1 / 4:5 / 9:16 を必須対応
- 共有前プレビューを必須にする
- 外部公開時は精密位置情報を出さない
- 画像生成は Cloud Functions 側を推奨
- OS共有シート中心で設計する
- アプリ内SNS化はしない

## 未決定事項
- QR / URL の表示可否
- 参加人数の表示可否
- デフォルトの位置表示粒度
- Map-first を既定にするか
- テンプレート自動推薦の有無

## 不足している証拠
- Instagram / LINE / X での実利用テスト結果
- 位置情報ぼかしの許容度に関するユーザーテスト
- どのテンプレートが最も共有率が高いかの定量データ

## リスク
- 地図情報の出しすぎによるプライバシー事故
- 文字を載せすぎてSNSで見栄えが崩れる
- 共有導線が複雑で使われない
- 生成コストが高くなる
- 位置情報の公開ルールが曖昧だと審査・運用リスクが高い

必要であれば次に、**この仕様をそのまま実装できるレベルの「画面仕様書」または「share_cards のデータモデル/API仕様」** に落とし込みます。
