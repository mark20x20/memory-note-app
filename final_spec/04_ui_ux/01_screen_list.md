# Screen List v2

## 1. 目的

Release v1 で必要な画面を、React Native + Expo 実装前提で整理します。  
前提は以下です。

- 写真を起点に思い出ノートを自動生成する
- iOS / Android 対応
- クラウド保存前提
- アカウント前提
- 共有ノート / 共同編集あり
- Owner / Editor / Viewer 権限あり
- SNS共有カード必須
- 権限説明、AI説明、位置情報説明を含む
- アプリ内SNSフィード、いいね、コメント、フォローは含めない

---

## 2. 画面分類

### Onboarding / Auth
初回説明、ログイン、アカウント作成を扱う画面。

### Home / Navigation
アプリの起点、ノート一覧、主要導線を扱う画面。

### Memory Note Creation
写真選択から思い出ノートを作成する画面。

### Photo Upload / Processing
アップロード進捗、失敗再試行、処理中状態を扱う画面。

### AI Generation / Edit
AI生成結果の確認、編集、再生成を扱う画面。

### Note Detail / Viewing
ノート詳細、写真、地図、メモ、メンバー閲覧を扱う画面。

### Map / Calendar / Search
地図、カレンダー、検索、振り返りを扱う画面。

### Sharing / Collaboration
招待、メンバー管理、権限管理、離脱を扱う画面。

### SNS Share Card
共有カード生成、プレビュー、保存、共有を扱う画面。

### Settings / Privacy / Support
設定、権限説明、プライバシー、利用規約、問い合わせを扱う画面。

### Error / Empty / Permission States
権限不足、空状態、失敗状態、復旧導線を扱う画面。

## 15. Bottom Navigation 構成

推奨構成は以下です。

- ホーム
- 地図
- 作成
- カレンダー
- 設定

### 理由
- 「写真から思い出ノートを作る」導線が中心
- 地図とカレンダーで振り返りやすい
- 作成を中央に置くことで主フローが明確
- 設定に権限・プライバシーを集約できる

### 調整案
| 案 | 内容 | メリット | デメリット |
|---|---|---|---|
| 案A: 標準案 | ホーム / 地図 / 作成 / カレンダー / 設定 | 分かりやすい | 検索入口が弱い |
| 案B: 検索重視 | ホーム / 地図 / 作成 / 検索 / 設定 | 探しやすい | カレンダー導線が弱い |
| 案C: 振り返り重視 | ホーム / 地図 / 作成 / カレンダー / 検索 | 回遊しやすい | 設定が奥まる |

### 推奨
- **案Aを基本**
- 検索はホーム上部に配置
- On This Day はホームとカレンダー内に配置
